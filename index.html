AI-Assisted Workflow —

Linear & Claude at Arkyn

A complete walkthrough of our process for shipping software with Linear and Claude.

The Big Picture

The Arkyn team uses Linear to track all work and Claude (through Claude Code, Cowork, and Chrome) to execute that work. The workflow integrates with Sentry, Zendesk, GitHub, Slack, XcodeBuildMCP, Dash Docs, and Scheduled Tasks to create a seamless system where issues become projects, projects become code, and code gets deployed.

Core principle: Let Claude read the issue in Linear, understand context from tools, and ship code without context-switching.

Our Linear Teams

We organize work into three teams in Linear:

Team Focus Platform Tools
Product Planning, design specs, research Web & Mobile Figma, Slack, Zendesk
Backend APIs, data, infrastructure Server GitHub, Sentry, AWS
Mobile iOS & Android clients iOS, Android GitHub, XcodeBuildMCP, Dash, TestFlight

Setup Overview

To get the full workflow running, you need:

  1. Claude Code — The IDE for working with projects
  2. Cowork — The plugin ecosystem for integrations
  3. Chrome Extension — Claude in your browser
  4. MCP Integrations — Tools like Slack, GitHub, Sentry, Dash, and more

Claude Code Setup

Claude Code is Anthropic's official CLI for Claude. It lets you open projects and work with them directly.

Installation:

npm install -g @anthropic-ai/claude-code

Usage:

claude-code open /path/to/project

Once opened, you can ask Claude to read files, run commands, make commits, and push code—all without leaving the tool.

Cowork Setup

Cowork is the plugin platform that extends Claude's capabilities. Plugins provide access to tools like Linear, Slack, GitHub, and more.

Install a plugin:

claude setup-cowork

This opens a guided setup flow where you can:

  • Choose your role (designer, engineer, product manager, etc.)
  • Select matching plugins
  • Authenticate with each tool
  • Test the connection

Once set up, your plugins persist and are available in all future Claude sessions.

Chrome Extension Setup

The Chrome extension brings Claude into your browser, so you can chat with Claude while browsing Linear, GitHub, Sentry, or any other web tool.

Install: Search for "Claude" in the Chrome Web Store and install.

Authenticate: Click the extension icon, log in with your Anthropic account, and grant the necessary permissions.

Usage: Highlight text, right-click "Ask Claude," and get responses in a side panel without leaving the page.

MCPs (Model Context Protocol) are how Claude connects to external tools. The key ones for Arkyn:

Integration Use Case When to Use
Linear Read/write issues, create cycles, fetch project context Planning, creating tasks, reading issue details
GitHub Create/update files, push commits, open PRs Coding, CI/CD, code review
Slack Send messages, create channels, read threads Notifications, team communication, announcements
Sentry Fetch error details, analyze crashes, create issues Debugging, root cause analysis, monitoring
Zendesk Read/write tickets, add comments, fetch user info Customer support, issue triage, responses
XcodeBuildMCP Build iOS/macOS apps, run tests, manage simulators Mobile development, CI/CD, device testing
Dash Search documentation offline, fetch code examples Learning APIs, checking syntax, reference lookup
Scheduled Tasks Run tasks on a schedule or manually, track progress Automation, periodic checks, recurring workflows

The Product Brief

A good issue includes:

What

What needs to be built? A one-sentence summary.

Why

Why is it important? Customer impact, OKR alignment, or business metric.

How

How might we do it? Acceptance criteria, rough approach, design specs.

Who

Who's involved? Platform badges, owner, blocked by, dependencies.

→

Ask Claude to read the issue

Claude writes code
→
Push PR to GitHub

What Claude Does

Dev

Reads the issue and understands the requirements.

Searches the codebase to find similar patterns and understand architecture.

Writes code in the correct style and format.

Runs tests to validate the implementation.

Commits and pushes to GitHub, with PR descriptions auto-generated from the branch name.

Branch Naming Convention

Include the Linear issue ID in the branch name:

git checkout -b DEV-123-add-user-auth

Pro Tip: The issue ID in the branch name helps Linear and GitHub stay in sync. When you push a branch, the PR is linked automatically, and when you merge, the issue closes.

Code Review

Claude can request a Copilot code review on your PR:

claude-code request-copilot-review [owner/repo PR#]

Copilot provides feedback on code quality, performance, and style. After addressing feedback, Claude pushes another commit.

Dependencies

In Linear, use blockedBy to mark dependencies:

  • iOS issue is blockedBy backend API issue
  • When backend issue closes, Linear notifies the iOS team
  • Claude can read dependencies and suggest parallel work

Testing & QA

iOSAndroidWeb

Testing is part of the development flow, not a separate step.

Test Writing

Claude writes unit tests alongside the code. For iOS:

XCTest // Unit tests
XCUITest // UI tests

For Android:

JUnit // Unit tests
Espresso // UI tests

For Web:

Jest // Unit tests
Playwright // E2E tests

Continuous Integration

Every PR runs tests on GitHub Actions. If tests fail, Claude updates the branch and commits a fix.

Device Testing (iOS)

For iOS, use XcodeBuildMCP to run tests on the simulator:

claude-code build-run-sim

Claude can:

  • Boot the simulator
  • Build the app
  • Run UI tests
  • Capture screenshots
  • Read logs

Sentry

Ops

Sentry tracks crashes and errors in production. Claude can read crash reports and create issues.

Workflow

Error occurs in production
→
Sentry captures it
→
Claude reads Sentry
→
Creates Linear issue
→
Team fixes it

What Claude Does

  • Fetches error details from Sentry
  • Runs root cause analysis (using Seer)
  • Creates a Linear issue with stack trace and reproduction steps
  • Assigns to the relevant team

GitHub

Dev

GitHub is where code lives. Claude pushes commits, creates PRs, and manages branches.

What Claude Does

  • Reads existing code to understand patterns
  • Creates branches with the Linear issue ID
  • Writes code and commits with clear messages
  • Opens PRs with auto-generated descriptions
  • Responds to review feedback and updates the branch

Pro Tip: Branch names like DEV-123-add-auth automatically link the PR to the Linear issue. GitHub and Linear stay in sync, and when you merge, the issue closes automatically.

Slack

PM

Slack is how the team communicates. Claude can read channels, post updates, and notify teams.

What Claude Does

  • Reads Slack threads to understand context
  • Posts status updates to channels
  • Notifies teams when issues are created or updated
  • Responds to messages in threads
  • Creates channels for projects

Note: Claude respects permissions and only posts to channels where it has access.

What Claude Does

  • Runs unit and UI tests
  • Captures screenshots

  • Installs apps on the simulator

claude-code build-run-sim
# Claude boots simulator, builds, and runs the app
# Then captures screenshots or runs tests
# Commits results back to the branch

Dash Docs

Dev

Dash is an offline documentation browser. Claude can search it for API references and code examples.

What Claude Does

  • Searches for documentation on iOS, Android, Web frameworks
  • Fetches code examples
  • Checks function signatures and return types
  • Looks up syntax without leaving Claude Code

Pro Tip: Dash is faster than online searches and doesn't require a network connection. Perfect for when you need quick reference lookups while coding.

Scheduled Tasks

Ops

Scheduled Tasks let Claude run periodic workflows automatically—like daily standups, weekly reports, or nightly builds.

What Claude Does

  • Runs tasks on a schedule (daily, weekly, monthly, or custom)
  • Executes one-time tasks at a specific time
  • Reads Linear issues and creates status reports
  • Sends Slack notifications
  • Generates metrics and dashboards

Example: Daily Standup

Schedule a task to run every morning at 9 AM:

claude-code schedule "Daily standup" "0 9 * * *"

Claude reads the team's Linear cycle, gathers today's tasks, and posts a summary to Slack.

Pro Tip: Use Scheduled Tasks to automate routine work like status checks, metrics collection, or release notes generation.

Daily Loop

9 AM: Standup

Claude reads Linear, Slack, and Sentry to generate a daily standup.

10 AM: Grab a Task

Pick an issue from the Linear cycle that's ready to build. Run claude-code open . and ask Claude to read the issue.

10 AM – 5 PM: Build

Claude writes code, runs tests, and pushes PRs. You review and provide feedback.

5 PM: Review PRs

Claude responds to code review feedback, updates the branch, and merges when approved.

5:30 PM: Handoff

Update Linear issue status, link to the merged PR, and move to the next issue.

Conventions & Tips

Commit Message Convention

Claude writes clear, conventional commit messages:

feat(auth): add email verification
fix(api): handle null user gracefully
test(payment): add coverage for refunds
docs: update API reference

Pro Tip: Include the Linear issue ID in the first line or body for automatic linking.

Pull Request Descriptions

Claude generates PR descriptions automatically following these conventions. You just need to make sure your branch name includes the issue ID.
